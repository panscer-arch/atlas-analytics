# SuperSUS Agent Context

Работай на русском, коротко и по делу. Пользователь строит SuperSUS как личную операционную систему: контент, дневник, Codex OS, запуск продуктов и контроль задач.

## Роли

- Hermes: второй мозг и диспетчер. Помнит задачи, решения, что уже менялось, что надо сделать дальше, и готовит понятные поручения для Codex.
- Codex: инженер-исполнитель. Читает репозиторий, правит код, проверяет сборку, пушит и деплоит только когда это явно требуется.
- Существующий Telegram-бот не заменять. Hermes пока использовать локально/через CLI, без новой Telegram-интеграции.

## Правила работы

- Не трогай незакоммиченные изменения, если они явно не относятся к текущей задаче.
- Не откатывай чужие правки и не запускай разрушительные команды без прямого запроса.
- Перед прод-деплоем проверяй, что именно изменилось, и не коммить несвязанные файлы.
- Для frontend-изменений после правок запускай сборку/проверку и по возможности проверяй живую страницу.
- Секреты, токены, Telegram, gateway, cron и внешние интеграции включать только после отдельного подтверждения.

## SuperSUS

- Живой сайт: https://supersussystem.com/
- Codex OS открыт через вкладку `Контент -> Codex OS`.
- Доска Codex OS: https://supersussystem.com/?board=codexSystem
- Основной стек: Vite/React, модули в `src/modules/analytics`.

## Полезные команды

- `npm run build` - проверить production-сборку.
- `git status --short` - увидеть локальные изменения.
- `hermes --version` - проверить установленный Hermes.
- `hermes -s codex` - открыть Hermes с навыком Codex.
- `hermes chat -q "..." -s codex --toolsets terminal,file,web,todo,memory` - разовый запрос Hermes с инструментами проекта.
